## 一、项目概述

海邻到家是一个面向社区便利店的智能收银与营销管理系统，V1.0 版本采用多端独立开发模式，核心目标是实现门店的数字化运营管理。

### 1.1 项目定位
- **面向对象**: 社区便利店（夫妻店、连锁便利店）
- **核心场景**: 线下收银 + 线上营销
- **目标用户**: 门店收银员、店长、总部管理人员

## 二、核心功能需求

### 2.1 收银台 APP

#### 功能列表
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 登录认证 | 店铺ID + 操作员密码 | P0 |
| 商品浏览 | 分类筛选、搜索、扫码添加 | P0 |
| 购物车 | 数量修改、删除、清空、挂单 | P0 |
| 会员识别 | 扫码/手机号识别会员 | P1 |
| 支付结算 | 现金/微信/支付宝 | P0 |
| 小票打印 | ESC/POS 指令打印 | P1 |
| 离线收银 | 网络中断时本地保存订单 | P1 |

#### 业务流程
```
商品选购 → 加入购物车 → 会员识别（可选）→ 结算支付 → 打印小票
```

#### 特殊功能
- **晚8点清货模式**: 20:00-23:00 全场8折
- **挂单/取单**: 临时挂起订单，稍后继续
- **钱箱控制**: 支付后自动开钱箱

### 2.2 小程序商城

#### 功能列表
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 商品展示 | 列表、详情、规格 | P0 |
| 购物车 | 添加、修改、结算 | P0 |
| 会员中心 | 登录、等级、积分 | P0 |
| 订单管理 | 下单、支付、退款 | P0 |
| **团购接龙** | 团长发起、实时参与 | P2 |

#### 业务流程
```
商品浏览 → 加入购物车 → 确认订单 → 支付 → 到店自提/配送
```

### 2.3 总部管理后台

#### 功能列表
| 模块 | 功能 | 优先级 |
|------|------|--------|
| 门店管理 | 新增、编辑、状态切换 | P0 |
| 商品管理 | 上架、下架、调价 | P0 |
| 订单管理 | 查询、退款处理 | P0 |

## 四、API 接口设计

### 4.1 认证接口
```
POST /api/auth/login          - 登录
POST /api/auth/logout         - 登出
GET  /api/auth/current        - 当前用户
```

### 4.2 商品接口
```
GET  /api/products            - 商品列表
GET  /api/products/:id        - 商品详情
POST /api/products/barcode    - 条码查询
```

### 4.3 订单接口
```
POST /api/orders              - 创建订单
POST /api/orders/:id/pay      - 支付
POST /api/orders/:id/cancel   - 取消
GET  /api/orders              - 订单列表
```

### 4.4 会员接口
```
POST /api/members/register    - 注册
POST /api/members/login        - 登录
POST /api/members/verify      - 验证
GET  /api/members/:id          - 详情
```

### 4.5 支付接口
```
POST /api/payments/create      - 创建支付
GET  /api/payments/query      - 查询状态
POST /api/payments/refund     - 退款
```

## 五、硬件集成

### 5.1 打印机
- **协议**: ESC/POS
- **功能**: 小票打印、标签打印
- **控制码**:
  - 初始化: `[ESC, 0x40]`
  - 居中: `[ESC, 0x61, 0x01]`
  - 切纸: `[GS, 0x56, 0x00]`

### 5.2 钱箱
- **触发**: 支付成功后自动弹开
- **指令**: `[ESC, 0x70, 0x00, 0x19, 0xFA]`

### 5.3 扫码枪
- **模式**: USB HID / 蓝牙 SPP
- **事件**: 扫码后自动填充商品搜索

## 六、V1.0 问题与改进

### 6.1 V1.0 存在的问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 代码重复 | 各端独立开发，共享逻辑未抽取 | 维护成本高 |
| 状态分散 | 各端独立管理状态，不同步 | 数据不一致 |
| 部署复杂 | 多端独立部署，流程繁琐 | 运维成本高 |
| 离线支持弱 | 仅收银台支持离线 | 用户体验差 |

### 6.2 V2.0 改进方向

| 改进点 | V1.0 | V2.0 |
|--------|------|------|
| 架构 | 独立开发 | Monorepo 统一架构 |
| 共享 | 复制粘贴 | packages 共享包 |
| 状态 | 分散管理 | Zustand 全局状态 |
| 部署 | 独立部署 | 统一部署平台 |
| 离线 | 仅收银台 | 全端离线支持 |

## 八、总结

V1.0 版本完成了海邻到家系统的核心功能验证，证明了业务模式的可行性。通过 V1.0 的实践，我们积累了以下经验：

1. **明确了业务边界**：收银、会员、营销三大核心
2. **验证了技术选型**：React + Taro + Next.js 组合可行
3. **发现了改进空间**：代码复用、状态管理需要优化

V2.0 架构将基于 V1.0 的经验，采用 Monorepo 模式重构，实现真正的代码复用和统一管理。
