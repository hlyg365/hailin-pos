# 海邻到家多终端系统连接方案 V1.0

## 一、系统架构总览

### 1.1 终端定位与用户角色

| 终端 | 路径 | 用户角色 | 使用场景 | 核心功能 |
|------|------|----------|----------|----------|
| **统一入口首页** | `/` | 所有用户 | 快速导航 | 各端入口、快捷操作 |
| **总部管理后台** | `/dashboard/*` | 总部管理员 | PC端办公 | 门店管理、商品管理、订单管理、供应链管理、报表分析 |
| **收银台APP** | `/pos/*` | 收银员 | 线下收银 | 商品扫码、收银结算、打印小票、会员识别 |
| **小程序商城** | `/mini-store/*` | 顾客 | 手机端购物 | 商品浏览、购物车、下单支付 |
| **店长助手PWA** | `/assistant/*` | 店长 | 移动办公 | 移动收银、库存盘点、报表查看、采购申请 |
| **门店管理PC端** | `/store-admin/*` | 店长 | PC端管理 | 门店数据、订单管理、促销管理 |

## 二、数据共享与同步策略

### 2.1 统一数据模型

| 数据类型 | 共享范围 | 同步策略 | 说明 |
|----------|----------|----------|------|
| **商品数据** | 全平台 | 实时同步 | 总部发布，各端共享 |
| **会员数据** | 全平台 | 实时同步 | 统一会员体系，跨端识别 |
| **订单数据** | 按门店隔离 | 实时同步 | 门店只能查看本门店订单 |
| **库存数据** | 按门店隔离 | 实时同步 | 各门店独立库存 |
| **促销数据** | 按配置 | 定时同步 | 总部配置后下发 |
| **优惠券数据** | 全平台 | 实时同步 | 线上线下共用 |

### 4.3 登录后自动跳转逻辑

```typescript
// 首页登录检测逻辑
const getLoginRedirect = () => {
  // 1. 检查各端登录状态
  const posUser = localStorage.getItem('pos_user');
  const dashboardUser = localStorage.getItem('dashboard_user');
  const assistantUser = localStorage.getItem('assistant_user');
  const storeAdminUser = localStorage.getItem('store_admin_user');

  // 2. 优先级判断
  if (posUser) return '/pos';           // 收银台优先
  if (storeAdminUser) return '/store-admin';  // 门店管理次之
  if (assistantUser) return '/assistant';    // 店长助手
  if (dashboardUser) return '/dashboard';    // 总部后台

  // 3. 全部未登录，返回首页
  return '/';
};
```

### 5.2 API统一封装

```typescript
// lib/api/client.ts
class UnifiedApiClient {
  private baseURL: string;
  private token: string | null;

  constructor() {
    this.baseURL = process.env.NEXT_PUBLIC_API_URL;
    this.token = null;
  }

  // 根据登录类型选择Token
  setToken(token: string, loginType: string) {
    this.token = token;
    localStorage.setItem(`${loginType}_token`, token);
  }

  // 通用请求方法
  async request<T>(config: ApiConfig): Promise<ApiResponse<T>> {
    // 添加对应类型的Token
    const headers = {
      'Content-Type': 'application/json',
      ...(this.token && { Authorization: `Bearer ${this.token}` }),
    };

    return this.httpRequest({ ...config, headers });
  }

  // 门店数据请求（自动带上门店ID）
  async storeRequest<T>(config: ApiConfig): Promise<ApiResponse<T>> {
    const storeId = this.getCurrentStoreId();
    return this.request<T>({
      ...config,
      url: `${config.url}?storeId=${storeId}`,
    });
  }
}

export const apiClient = new UnifiedApiClient();
```

### 5.3 跨端数据同步

```typescript
// hooks/useDataSync.ts
export function useDataSync(dataType: 'products' | 'orders' | 'members') {
  const [data, setData] = useState([]);
  const [syncing, setSyncing] = useState(false);

  // 实时同步
  useEffect(() => {
    const channel = new BroadcastChannel('data-sync');
    
    channel.onmessage = (event) => {
      if (event.data.type === dataType) {
        setData(event.data.payload);
      }
    };

    return () => channel.close();
  }, [dataType]);

  // 推送更新
  const sync = useCallback(async () => {
    setSyncing(true);
    try {
      const freshData = await fetchData(dataType);
      setData(freshData);
      
      // 通知其他标签页
      const channel = new BroadcastChannel('data-sync');
      channel.postMessage({ type: dataType, payload: freshData });
    } finally {
      setSyncing(false);
    }
  }, [dataType]);

  return { data, syncing, sync };
}
```

- 访问 `/download` 下载APK
